# iOS Shortcut Setup — Inkcast

This guide covers three shortcut configurations, from simplest to most powerful.

---

## Shortcut 1 — Quick Launch (Recommended)

Opens Inkcast in Safari with one tap. Add it to your Home Screen for an app-like experience.

**Actions:**

| Step | Action | Setting |
|---|---|---|
| 1 | Open URLs | `https://<your-username>.github.io/inkcast/` |

That's it. One action.

**Setup:**
1. Open Shortcuts app → tap **+**
2. Search for **"Open URLs"** → add it
3. Paste your Inkcast URL
4. Tap the shortcut name → rename to **Inkcast**
5. Tap the icon → choose 📖 or any book icon
6. Tap **Done**
7. Long-press the shortcut → **Add to Home Screen**

---

## Shortcut 2 — Siri Launch

Say "Hey Siri, Inkcast" to open it hands-free.

Same setup as Shortcut 1, but make sure the shortcut name is exactly what you want to say to Siri.

Good names: `"Inkcast"`, `"Open my books"`, `"Read to me"`

---

## Shortcut 3 — Lock Screen / Control Center Button

1. Go to **Settings → Accessibility → Accessibility Shortcut** (or Control Center)
2. Or: **Settings → Lock Screen** → add a Shortcuts widget
3. Select your Inkcast shortcut

## Tips

**Offline use:**
After visiting Inkcast once in Safari, it caches in your browser. But epub files are loaded fresh each time — keep your epubs in the Files app (iCloud Drive works great) for easy access.

**Speed tip:**
If you use 1.5× or 2× speed often, that setting is remembered within a session but resets on reload. A future update may persist it.

**Best voices for listening:**
On iOS, go to **Settings → Accessibility → Spoken Content → Voices → English** and download:
- **Siri Voice 1 or 2** (most natural)
- **Karen** (Australian, clear)
- **Daniel** (British, excellent for fiction)

Downloaded voices appear in Inkcast's voice picker automatically.

---

## Troubleshooting

**"Speech not supported"** — Make sure you're using Safari, not an in-app browser.

**Book doesn't parse** — Some epub files use non-standard structure. Try opening the epub in a different reader first to confirm it's valid.

**Shortcut opens a blank page** — GitHub Pages can take a few minutes to go live after first setup. Try again after 2 minutes.
